# 安全规约（SR-security-rules.md）

> 全阶段加载，所有角色强制遵守。

---

## 认证与传输安全

| 编号      | 规约内容                                     | 违反处置 |
| ------- | ---------------------------------------- | ---- |
| SR-0001 | 认证接口不允许明文传输密码，必须使用 HTTPS + 加密传输          | 阻断   |
| SR-0002 | 敏感信息（密码、Token、密钥）不允许明文记录在文档、注释、日志中       | 阻断   |
| SR-0003 | .env/.pem/.key/credentials 等敏感文件禁止提交 git | 阻断   |
| SR-0004 | JWT/Session Token 必须设置过期时间，不允许永不过期       | 阻断   |

---

## 输入验证与注入防护

| 编号      | 规约内容                       | 违反处置 |
| ------- | -------------------------- | ---- |
| SR-0005 | 所有用户输入必须在服务端做校验，不得仅依赖前端校验  | 阻断   |
| SR-0006 | 数据库操作必须使用参数化查询，禁止字符串拼接 SQL | 阻断   |
| SR-0007 | 输出到页面的内容必须做 XSS 转义处理       | 阻断   |
| SR-0008 | 文件上传必须校验文件类型和大小，禁止直接执行上传文件 | 阻断   |

---

## 权限与访问控制

| 编号 | 规约内容 | 违反处置 |
|------|---------|---------|
| SR-0009 | 所有 API 接口必须做权限校验，禁止未授权访问 | 阻断 |
| SR-0010 | 管理员接口必须与普通用户接口分离，独立鉴权 | 阻断 |
| SR-0011 | 敏感操作（删除、修改权限）必须记录操作日志，含操作人和时间 | 警告 |

---

## 环境与配置安全

| 编号 | 规约内容 | 违反处置 |
|------|---------|---------|
| SR-0012 | 开发/测试/生产环境必须使用不同的密钥和数据库 | 阻断 |
| SR-0013 | 生产环境禁止开启 DEBUG 模式 | 阻断 |
| SR-0014 | 第三方依赖必须锁定版本，定期扫描已知漏洞（bandit/safety） | 警告 |
| SR-0015 | Docker 镜像禁止使用 root 用户运行业务进程 | 警告 |
