# 设计规约（DR-design-rules.md）

## 架构分层规范

| 层次                | 职责                        | 禁止事项      |
| ----------------- | ------------------------- | --------- |
| Controller/View 层 | 接收请求、参数校验、调用 Service、返回响应 | 禁止包含业务逻辑  |
| Service 层         | 业务逻辑编排、事务管理               | 禁止直接操作数据库 |
| Repository/DAO 层  | 数据库操作、查询封装                | 禁止包含业务逻辑  |
| Model 层           | 数据结构定义                    | 禁止包含业务逻辑  |
| 胶水层设计             | 层与层连接处必须有明确的胶水层设计         | 禁止隐式耦合    |
| 页面-路由-接口契约        | 前端页面、路由、后端接口之间必须有明确的契约定义  | 禁止无契约     |

---

## 数据库设计规范

| 项目 | 规范内容 |
|------|---------|
| 表命名 | 小写下划线，复数形式，如 user_accounts |
| 字段命名 | 小写下划线，如 created_at、updated_at |
| 必备字段 | 每张表必须有 id（主键）、created_at、updated_at |
| 软删除 | 需软删除的表加 deleted_at 字段 |
| 外键 | 必须在设计文档中声明外键关系，代码中可选择是否建物理外键；是否建物理外键由架构师在 DESN 文档中逐表声明，未声明则默认不建。 |
| 索引 | 查询条件字段、外键字段必须建索引 |

---

## 接口设计规范

| 项目 | 规范内容 |
|------|---------|
| 风格 | RESTful，资源名用复数，如 /api/v1/users |
| 版本 | URL 路径版本号，如 /api/v1/ |
| 响应格式 | 统一 JSON：{"code": 0, "msg": "ok", "data": {...}} |
| 错误码 | 业务错误码从 1000 开始，文档中列出所有错误码 |
| 分页 | 统一使用 page + page_size 参数 |

---

## DESN 文档规范

| 项目     | 规范内容                                |
| ------ | ----------------------------------- |
| 命名规范   | docs/DESN-{项目简称}-v{版本号}.md          |
| 状态管控   | 草稿 → 待审批 → 已批准，状态写在文档头部             |
| 内容边界   | DESN 描述"怎么做"，必须覆盖 PRD 中所有功能点        |
| CHK 文档 | 必须同步生成 CHK 一致性检查文档，逐项对照 PRD vs DESN |
| 红线     | 已批准的 DESN 不得直接修改，变更需走变更流程           |
