## 数据安全类

| 编号 | 红线内容 | 级别 |
|------|---------|------|
| RL-DV-0005 | 开发/测试数据库必须分离，禁止用生产库做测试 | 阻断 |
| RL-TS-0001 | 测试环境必须与生产环境配置一致（OS、中间件版本） | 阻断 |

## 流程管控类

| 编号         | 红线内容                                                                    | 级别  |
| ---------- | ----------------------------------------------------------------------- | --- |
| RL-MG-0001 | 文档状态未达"已批准"不得推进下一阶段                                                     | 阻断  |
| RL-MG-0002 | 里程碑准出检查未通过不得更新 CLAUDE.md 当前阶段                                           | 阻断  |
| RL-RQ-0003 | 需求变更必须正式记录到 CHANGELOG-{项目}.md，格式：\| 日期 \| 变更描述 \| 影响范围 \| 发起人 \|，不得口头变更 | 阻断  |
| RL-DV-0004 | 任务状态必须反映真实情况，禁止虚报完成                                                     | 阻断  |
| RL-DV-0005 | **禁止跳过阶段**: 必须按 M1 → M2 → M3 → M4 → M5 → M6 顺序推进                        | 阻断  |
| RL-RQ-0006 | 已批准的 PRD 不得直接修改，需记录需求变更情况，经用户再次批准                                       | 阻断  |

## 代码管理类

| 编号         | 红线内容                                                       | 级别  |
| ---------- | ---------------------------------------------------------- | --- |
| RL-DV-0001 | 不允许在 main/master 分支直接开发，文档编写不受限                            | 阻断  |
| RL-DV-0002 | 代码修改完成必须提交 git，不允许长期未提交                                    | 警告  |
| RL-DV-0003 | 单元测试未全部通过不得提交 git                                          | 阻断  |
| RL-DV-0007 | 静态代码扫描存在阻断级别问题不得合并                                         | 阻断  |
| RL-DV-0008 | 业务代码开发必须在 worktree 隔离工作区中进行，禁止在主工作区直接编写业务代码（文档编写、配置修改不受此限） | 阻断  |

## 验证与声明类

| 编号         | 红线内容                                                          | 级别  |
| ---------- | ------------------------------------------------------------- | --- |
| RL-VF-0001 | 声明完成前必须有新鲜的验证证据（在本次 subagent 会话内亲自运行命令并展示输出），禁止引用历史结果或未验证声明完成 | 阻断  |
| RL-VF-0002 | 禁止使用"should work"、"probably fixed"等未验证表述                      | 警告  |
| RL-VF-0003 | 禁止使用"let me try this and see"（猜测性修复）                          | 警告  |
