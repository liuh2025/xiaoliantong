# 业务规约库（BR-business-rules.md）

## 使用说明
- 本规约为通用业务规则积累库，适用于无特殊业务要求的场景
- 若业务需求与本规约冲突，以业务需求为准，并在 PRD 中注明"覆盖 BR-xxx"
- 随着项目推进，持续积累新的业务规则到本文件

---

## 通用交互规约

| 编号     | 规约内容                          | 通用性 |
| ------ | ----------------------------- | --- |
| BR-001 | 新增/修改/删除操作需二次确认弹窗             | 通用  |
| BR-002 | 操作失败需给出具体错误原因，不得仅显示"操作失败"     | 通用  |
| BR-003 | 列表页默认按创建时间倒序排列                | 通用  |
| BR-004 | 分页默认每页 20 条，支持切换 10/20/50/100 | 通用  |

---

## 数据约束规约

| 编号     | 规约内容                     | 通用性 |
| ------ | ------------------------ | --- |
| BR-005 | 唯一约束键（如编码、名称）不支持修改，需删除重建 | 通用  |
| BR-006 | 被引用数据不可删除，需先解除引用关系       | 通用  |
| BR-007 | 软删除优先于物理删除，保留数据可追溯性      | 通用  |

## 账号与权限规约

| ------ | ----------------------- | --- |

| BR-009 | 登录错误提示需区分"用户不存在"和"密码错误" | 通用  |

## PRD 文档规范

### 命名规范
`docs/PRD-{项目简称}-v{版本号}.md`，版本号从 1.0 开始

### 状态管控
PRD 文档头部必须标注状态：
- 草稿：正在编写中
- 待审批：已完成，等待用户审批
- 已批准：用户已批准，可进入下一阶段
- 已归档：项目全部完成后

### 内容边界
- PRD 只描述"做什么"，不描述"怎么做"
- 内容严格按照PRD模板编写
- 技术实现细节写在 DESN 设计文档中
