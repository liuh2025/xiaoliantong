# 测试规约（TR-test-rules.md）

## 两层测试体系

| 层次        | 范围                                 | 工具                | 执行者             | 编写阶段    |
| --------- | ---------------------------------- | ----------------- | --------------- | ------- |
| L1 单元测试   | 单个函数/类的隔离测试                        | pytest            | 开发人员            | M3 开发阶段 |
| L1 API 测试 | 所有接口（RPC/RESTful/MQ）正例、反例、边界值、业务规则 | pytest + requests | Tester subAgent | M4 测试阶段 |
| L2 E2E 测试 | 完整用户操作流程，主流程/核心功能 100%，含异常测试       | Playwright        | Tester subAgent | M4 测试阶段 |

### 测试层级说明

- **L1 单元测试**：开发人员在 M3 阶段按 TDD 方式编写，准出前必须 100% 通过
- **L1 API 测试**：测试人员在 M4 阶段编写脚本并执行，覆盖所有接口类型
- **L2 E2E 测试**：测试人员在 M4 阶段编写脚本并执行，从用户视角验证业务流程

---

## 测试用例规范

| 项目   | 规范内容                                        |
| ---- | ------------------------------------------- |
| 用例编号 | TC-{层级}-{类型}-{三位序号}，如 TC-API-001、TC-E2E-005 |
| 用例内容 | 必须包含：前置条件、操作步骤、预期结果                         |
| 覆盖要求 | 正例 + 反例 + 边界值 + 业务规则验证                      |
| 先写用例 | 必须先在 QA-test-plan 中编写用例，再编写测试脚本             |

---

## 测试数据规范

| 项目 | 规范内容 |
|------|---------|
| 数据可重复 | 每次测试前重置测试数据，确保用例可重复执行 |
| 数据预置 | 测试数据通过 fixtures 或 setup 脚本预置，不得手动录入 |

---

## E2E 截图规范

| 项目 | 规范内容 |
|------|---------|
| 截图时机 | 功能响应后截图（成功：成功界面；失败：错误提示界面） |
| 命名格式 | {用例ID}-{结果}[-{详情}].png，如 TC-E2E-001-成功.png、TC-E2E-001-失败-密码错误.png |
| 存放位置 | tests/e2e/screenshots/{YYYY-MM-DD_HHMMSS}/ 目录 |
| 覆盖要求 | 每个 E2E 用例必须有截图 |

---

## 测试报告规范

| 项目 | 规范内容 |
|------|---------|
| 命名规范 | docs/TEST-REPORT-{项目简称}-v{版本号}.md |
| 准出结论 | 报告末尾必须有 QA 签署的审查结论 |

---

## 冒烟测试规范

| 项目 | 规范内容 |
|------|---------|
| 用例选择 | 从 L2 E2E 用例中选择核心路径用例作为冒烟测试 |
| 执行时机 | M5 部署阶段，部署完成后立即执行 |
| 通过标准 | 所有冒烟测试用例必须 100% 通过 |
| 标记方式 | 使用 `@pytest.mark.smoke` 标记冒烟测试用例 |

## 目录结构

```
tests/
├── unit/                        # L1 单元测试（M3 开发阶段）
│   └── {module}/
│       └── test_*.py
│
├── integration/                 # L1 API 测试（M4 测试阶段）
│   ├── script/                  # 测试脚本
│   └── log/                     # 执行日志（按时间分目录）
│
├── e2e/                         # L2 E2E 测试（M4 测试阶段）
│   ├── script/                  # 测试脚本（Page Object 模式）
│   ├── screenshots/             # 测试截图（按时间分目录）
│   └── log/                     # 执行日志（按时间分目录）
│
├── screenshots/                 # 冒烟测试截图（M5）
│   └── smoke/
└── test-data/                   # 测试数据
```

---

## 执行命令

```bash
# L1 单元测试（M3 开发阶段）
pytest tests/unit/ -v --cov=src --cov-report=html

# L1 API 测试（M4 测试阶段）
pytest tests/integration/script/ -v

# L2 E2E 测试（M4 测试阶段）
pytest tests/e2e/script/ -v --headed

# M5 冒烟测试
pytest tests/e2e/script/ -v -m smoke --headed
```
