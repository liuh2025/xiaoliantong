## 1. 项目概述

### 1.1 项目背景

在当前汽车产业加速变革的背景下，供应链的稳定与高效寻源成为企业核心竞争力。燕山大学汽车学院校友会供应链分会汇聚了大量汽车产业链上下游的优秀校友企业，具备天然的信任基础与合作潜能。然而，校友企业间仍面临“信息不对称、匹配效率低”的痛点。现有的“燕企通”功能相对单一，结构化程度不够。基于此，“校链通”应运而生，作为燕企通的升级替代版。

### 1.2 项目目标

本项目一期（MVP版本）的核心目标是“快速上线，验证场景”，跑通“供需信息发布-检索-交换联系方式”的核心撮合场景，验证校友间商业互信的业务价值。

### 1.3 项目范围

**In Scope (一期 MVP)**：
- **宣传页：** 统一宣传页+二维码入口，用于校友推广
- 核心业务：企业名录、商机广场（供需发布与单向撮合）、校友圈互动、消息通知。
- **企业端管理中心：** 企业信息维护、本企业员工账号管理、本企业商机管理。
- **平台端管理中心：** 数据大盘、平台企业(租户)管理、平台企业入驻审核、平台商机管理、基础数据字典、账号与权限管理、系统设置。

**Out of Scope (一期排除项)**：
- 深度的业务协同（如：询报价流转、订单执行、财务对账）。
- 燕大”东重人”系统的自动化对接认证。
- 企业私有宣传页（本期为统一宣传页）

## 2. 用户角色与权限

### 2.1 角色定义

| 角色              | 权限范围                                                |
| --------------- | --------------------------------------------------- |
| **超级管理员 (平台方)** | 平台端管理中心的所有权限（数据大盘、平台企业管理、商机审核、权限配置等）                |
| **平台运营 (平台方)**  | 平台端管理中心的内容审核与基础数据维护，不接触系统底层的账号和权限配置                 |
| **企业管理员 (企业端)** | 前台业务浏览与发布；企业端管理中心权限（认领企业、维护本企业信息、管理本企业员工账号、管理本企业商机） |
| **普通员工 (企业端)**  | 前台业务浏览与发布（浏览商机、发布供需、发起沟通、参与校友圈互动）；个人信息设置            |
| **注册游客 (未认证)**  | 仅能浏览部分公开脱敏信息，无权发布商机和获取联系方式。可发起“认领/创建企业”申请。          |

### 2.3 账号注册与角色转化链路
1. **游客注册**：通过手机号+短信验证码在首页进行快速注册登录，初始角色为【注册游客】。
2. **升级为企业管理员**：游客在“企业名录”发起认领或创建企业申请。平台后台审核通过后，该用户账号 (`user_id`) 与该企业 (`enterprise_id`) 绑定，角色自动升级为【企业管理员】。
3. **升级为普通员工**：由【企业管理员】在企业工作台中通过“邀请/新增员工”录入手机号。
   - 若该手机号已是注册游客，直接绑定企业并升级为【普通员工】。
   - 若未注册，系统自动为其创建初始账号（密码默认手机号后6位），并分配为【普通员工】，该员工首次登录需强制修改密码。

## 3. 功能需求

### 3.1 首页与导航

#### 3.1.1 业务描述
首页作为用户进入平台的第一触点，承担着全局导航、核心数据展示、快捷入口分发以及优质内容推荐的作用。通过合理的页面布局，引导用户快速感知平台生态（商机、企业、动态），并便捷进入发布或检索流程。

#### 3.1.2 界面元素与数据字段

| 字段名                 | 字段类型    | 说明                 | 来源                      |
| ------------------- | ------- | ------------------ | ----------------------- |
| total_enterprises   | INT     | 累计入驻企业数量           | 系统统计 (`enterprises`表)   |
| total_opportunities | INT     | 累计发布商机数量           | 系统统计 (`opportunities`表) |
| successful_matches  | INT     | 成功撮合对接数            | 系统统计 (撮合记录表)            |
| active_alumni       | INT     | 活跃校友人数             | 系统统计 (系统用户表近期活跃)        |
| search_keyword      | VARCHAR | 全局搜索关键词            | 用户输入                    |
| hot_tags            | ARRAY   | 热门检索标签列表（行业/品类/地域） | 后台配置提取                  |
| new_enterprises     | ARRAY   | 新入驻的校友企业列表缩略展示     | 最近认领/入驻的企业记录            |
| latest_feeds        | ARRAY   | 最新校友动态缩略展示         | `feeds` 表最新记录           |

#### 3.1.3 操作流程

**全局搜索流程**：
```
用户在顶部搜索框输入关键词 → 按回车或点击搜索图标 → 
系统根据关键词匹配商机标题、企业名称、校友动态 → 
跳转至统一搜索结果页，分 Tab (找商机/找企业/看动态) 展示结果
```

**快捷发布跳转流程**：
```
用户点击“发布采购需求”或“发布供应能力”按钮 → 
系统判断用户是否已登录及是否绑定企业 → 
满足条件则直接弹出对应的发布弹窗（默认选中对应类型） →
弹窗内提交数据，进入商机发布流程
```

#### 3.1.4 业务规则

| 规则编号      | 规则描述                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| BR-HOM-01 | **快捷发布权限校验**：只有已登录且归属于某个认证企业的用户才能点击快捷发布按钮；未登录跳登录页，未绑定企业提示需先加入或认领企业。                                                                                                                                                                                                                                                                                                                                                                            |
| BR-HOM-02 | **热门检索规则**：后台运营管理界面可配置。最多配置 6 个标签。用户点击任一标签后，跳转至“商机广场”，并默认应用该标签进行商机检索。<br>**自动提取策略**：系统每日凌晨根据历史数据自动将使用频次高的标签提取到候选池。<br>**人工干预策略**：平台运营人员在后台可从候选池勾选需要展示在首页的热门标签，亦可手动增加系统字典项作为强力推荐。                                                                                                                                                                                                                                                               |
| BR-HOM-03 | **智能推荐规则**：展示 4 条优质商机卡片，推荐逻辑分两级：<br>**1. 游客或未绑定企业用户（冷启动推荐）**：优先展示全平台最新发布（按时间倒序）、状态为“ACTIVE”、且包含热门标签（`tags` 字段与 `hot_tags` 交集多）的商机。<br>**2. 已绑定认证企业用户（个性化智能匹配）**：基于该用户所属企业的属性进行多维度加权计算：<br> - **供需互补匹配 (高权重)**：若用户企业主营“新能源汽车”，则优先推荐其他企业发布的采购“新能源汽车”零部件的“采购需求(BUY)”，或者推荐上游材料的“供应能力(SUPPLY)”。<br> - **品类标签重合度匹配 (中权重)**：计算商机 `category_id` 和 `tags` 与该用户企业属性的重合度。<br> - **地域就近匹配 (低权重)**：优先推荐 `region_id` 相同或相近（如长三角）的商机。<br>综合得分 Top 4 的商机在首页展示。 |
| BR-HOM-04 | **最新企业展示规则**：展示最近 3 家完成认证（`auth_status` 为 VERIFIED）的校友企业 Logo 和名称及主营领域。点击卡片跳转到该企业的主页。                                                                                                                                                                                                                                                                                                                                                          |
| BR-HOM-05 | **最新动态展示规则**：展示最近 2 条校友圈动态（包含发布人、发布时间、摘要）。点击可跳转至校友圈详情。                                                                                                                                                                                                                                                                                                                                                                                         |

**操作流程**：
```
用户输入关键词 → 敲击回车或点击搜索图标 →

按相关性（标题优先匹配 > 内容匹配）排序 →
跳转至搜索结果页，默认展示"找商机" Tab
```

### 3.3 企业名录 (找企业)

#### 3.2.1 业务描述
构建可信的校友企业名录，集中展示企业基本信息和权威资质。支持企业主认领自己的企业，以获得企业主页的管理权限，并聚合展示该企业的所有供需商机。通过本模块，平台完成企业实体的认领或创建，建立企业的管理员账号体系。

#### 3.2.2 界面元素与数据字段 (`enterprises` 实体)

| ------------- | ------- | --- | --- | ------------------------------------------------- |
| id            | BIGINT  | Y   | Y   | 主键，自增                                             |
| name          | VARCHAR | Y   | Y   | 企业全称                                              |
| credit_code   | VARCHAR | Y   | Y   | 统一社会信用代码（18位）                                    |
| legal_representative | VARCHAR | Y | N | 法人姓名                                             |
| logo_url      | VARCHAR | N   | N   | 企业Logo地址                                          |
| industry_id   | BIGINT  | Y   | N   | 所属行业ID                                            |
| category_id   | BIGINT  | N   | N   | 主营业务品类ID                                          |
| region_id     | BIGINT  | Y   | N   | 所在地区ID                                            |
| tags          | JSON    | N   | N   | 供需标签数组 (如: 专精特新, 车规级)                             |
| description   | TEXT    | N   | N   | 企业简介                                              |
| business_license | VARCHAR | Y | N | 营业执照URL（认证必填）                                  |
| auth_status   | VARCHAR | Y   | N   | 认证状态: UNCLAIMED(未认领), PENDING(待审核), VERIFIED(已认证) |

#### 3.2.3 操作流程

**企业检索流程**：
```
用户进入“企业名录”页面 → 
通过顶部组合筛选栏选择全部行业、全部品类、所在地域（多层级下拉），或点击热门标签 → 
系统根据条件（AND 关系）查询企业的 `tags`, `industry_id`, `region_id` 等字段 → 
分页展示企业名片列表
```

**企业认领流程（针对系统已预置的未认领企业）**：
```
校友扫描宣传页二维码 → 进入注册页完成用户注册（手机号+验证码） →
登录后进入”企业名录”页面 → 点击未认领企业”认领该企业” →
弹出认领表单弹窗（企业名称和基本信息已自动带入且锁定） →
用户填写法人姓名、上传营业执照（必填）、申请人职务 → 点击提交 →
系统生成认领申请记录，该企业 `auth_status` 变更为 PENDING，等待后台审核
```

**企业创建流程（针对系统未收录的企业）**：
```
校友扫描宣传页二维码 → 进入注册页完成用户注册（手机号+验证码） →
登录后在”企业名录”页面点击”创建我的企业” → 弹出完整创建表单弹窗 →
用户填写企业全称、统一社会信用代码（必填）、法人姓名（必填）、所属行业、主营品类、所在地区、企业简介 →
上传营业执照（必填）、申请人职务 → 点击提交 →
系统新增一条企业记录，`auth_status` 初始状态为 PENDING，等待后台审核
```

#### 3.2.4 业务规则

| 规则编号      | 规则描述                                                                                                                  |
| --------- | --------------------------------------------------------------------------------------------------------------------- |
| BR-ENT-01 | **浏览与认领权限**：必须是认证校友登录后才可浏览企业详细信息和发起认领/创建；未登录或未认证用户点击时，引导其完成登录/认证，否则仅显示脱敏列表。                                           |
| BR-ENT-02 | **认领互斥机制**：一个企业只能被一个用户成功认领。如果企业 `auth_status` 已为 PENDING 或 VERIFIED，则前端隐藏或禁用认领按钮。                                     |
| BR-ENT-03 | **管理员权限分配**：后台审核通过企业认领或创建申请后，自动将申请人的系统角色变更为”企业管理员”，并将其 user_id 绑定至企业的 `admin_user_id` 字段，该企业状态变为 VERIFIED。            |
| BR-ENT-04 | **冷启动机制**：平台上线前由运营人员预置一批真实的汽车产业链企业数据，状态为 UNCLAIMED，供后续校友认领。                                                           |
| BR-ENT-05 | **企业认证必填字段**：统一社会信用代码（credit_code）、法人姓名（legal_representative）、营业执照（business_license）为认证必填字段，审核时须校验营业执照与企业名称、信用代码的一致性。 |

### 3.4 企业端管理中心 (我的企业)

#### 3.3.1 业务描述
企业端管理中心是前台系统的专属区域，仅对”企业管理员”角色开放。在这里，企业管理员可以维护企业主页对外的展示信息，管理本企业的员工账号体系，以及集中处理本企业发布的所有商机。这是平台实现租户级分层管理的核心模块。

#### 3.3.2 核心模块与功能

**1. 企业信息维护**
- **功能描述**：企业管理员可以修改本企业的基础信息，这些信息将直接同步到”企业名录”和企业主页中。
- **可修改字段**：`logo_url`、`description` (企业简介)、`tags` (供需标签)。
- **不可修改字段**：`name` (企业全称)、`credit_code` (统一社会信用代码)、`legal_representative` (法人姓名)、`industry_id` 等核心认证字段，若需修改须联系平台运营处理。

**2. 本企业员工账号管理**
- **功能描述**：企业管理员管理归属于本企业（`enterprise_id` 一致）的员工账号。
- **操作项**：
  - **邀请/新增员工**：输入员工姓名和手机号，分配“普通员工”或“企业管理员”角色，系统自动为其创建账号并绑定当前企业。
  - **停用账号**：一键停用离职或违规员工的账号，该账号将无法登录。
  - **重置密码**：将员工密码重置为默认值（如手机号后6位）。

**3. 本企业商机管理**
- **功能描述**：集中展示本企业内所有员工发布的所有采购需求和供应能力。
- **操作项**：
  - **查看详情**：查看各商机的详细内容、状态（ACTIVE/OFFLINE）和浏览量。
  - **主动下架**：对于已结束或不需要的商机，企业管理员可手动将其置为 OFFLINE，停止在广场展示。
  - **修改/重新发布**：编辑修改已有的商机内容。

### 3.5 商机广场 (供需发布与撮合)

#### 3.4.1 业务描述
商机广场是平台的核心流量池，解决校友企业间“找不到、找不准”的痛点。支持用户快速发布采购需求或供应能力，支持多维复合检索，并实现单向获取联系方式的极速撮合。

#### 3.4.2 界面元素与数据字段 (`opportunities` 实体)

| 字段名 | 字段类型 | 必填 | 唯一 | 说明 |
|--------|----------|------|------|------|
| id | BIGINT | Y | Y | 主键，自增 |
| type | VARCHAR | Y | N | 商机类型: BUY(我要买/采购需求), SUPPLY(我能供/供应能力) |
| title | VARCHAR | Y | N | 商机标题 |
| enterprise_id | BIGINT | Y | N | 发布企业ID (外键关联 enterprises 表) |
| publisher_id | BIGINT | Y | N | 发布人用户ID |
| industry_id | BIGINT | Y | N | 所属行业ID |
| category_id | BIGINT | Y | N | 所属业务品类ID |
| region_id | BIGINT | Y | N | 所在地区ID |
| tags | JSON | N | N | 业务标签数组 (支持多选) |
| detail | TEXT | Y | N | 详细描述/需求说明 |
| status | VARCHAR | Y | N | 状态: ACTIVE(生效展示中), OFFLINE(违规下架/主动关闭) |
| view_count | INT | Y | N | 浏览量，默认0 |

#### 3.4.3 操作流程

**商机检索流程**：
```
用户进入“商机广场”页面 → 
通过左侧栏选择商机类型（我要买/我能供） → 
通过左侧下拉组件选择行业、业务品类、所在地域 → 
系统根据条件动态更新右侧的商机列表（卡片式展示，包含标题、类型、标签、企业、时间、地点）
```

**商机发布流程**：
```
用户点击“发布商机”按钮 → 
弹出发布表单 → 
用户选择类型，填写标题，通过多级联动下拉框选择行业/品类/地区，选择/输入多个业务标签，填写详细描述 → 
点击提交 → 
系统校验数据完整性 → 
保存数据，状态默认为 ACTIVE，直接在广场展示
```

**获取联系方式撮合流程**：
```
意向方用户（已登录）在商机卡片点击“交换/获取联系方式” → 
弹出模态框提示即将获取对方联系方式 → 
用户确认获取 → 
系统展示发布方企业的联系人（publisher_id 对应的用户）姓名和电话/微信 → 
系统生成一条通知消息，发送给发布方（“某某企业-某某查看了您的联系电话”） →
系统记录一条撮合日志
```

#### 3.4.4 业务规则

| 规则编号      | 规则描述                                                                                                   |
| --------- | ------------------------------------------------------------------------------------------------------ |
| BR-OPP-01 | **发布审核机制（先发后审）**：用户发布商机成功后，`status` 直接置为 ACTIVE 并在广场展示，不需要人工审核。后台运营人员在管理中心进行事后巡查，发现违规直接下架（置为 OFFLINE）。 |
| BR-OPP-02 | **发布权限限制**：只有已登录、且所属企业已通过认证（即属于某租户）的用户，才能发布商机。                                                         |
| BR-OPP-03 | **获取联系方式（单向获取）**：意向方点击获取后，无需发布方同意，即可直接查看联系方式。但系统**必须**发消息通知发布方。                                        |
| BR-OPP-04 | **联系方式掩码**：在未点击“获取联系方式”前，商机卡片及详情中不允许展示完整的电话号码或微信号（使用 * 号掩码或隐藏）。                                        |

##### 3. 企业(租户)管理 (Tenants)
管理已通过审核的入驻企业（租户）及其挂载的员工账号。

**业务描述与数据结构**：
企业相当于一个“租户”，企业下的员工账号是该租户的“成员”。系统需要维护企业（`enterprises`表）与用户（`sys_users`表）的一对多关系。

**操作流程**：
1. **企业基础管理**：管理员查看所有企业列表 → 可执行 [停用企业] 操作（停用后，该企业下所有成员无法登录，该企业所有发布的商机自动下架）。

   - **新增成员**：管理员可手动为该企业添加员工账号（输入手机号、姓名），并分配系统角色（普通员工或企业管理员）。

#### 3.6.3 核心数据结构 (`sys_users` / `sys_roles`)

| 字段名 | 字段类型 | 说明 |
|--------|----------|------|
| id | BIGINT | 用户主键 |
| username | VARCHAR | 登录账号/手机号 |
| password | VARCHAR | 加密密码 |
| real_name | VARCHAR | 真实姓名 |
| enterprise_id | BIGINT | 归属企业(租户)ID，为 Null 表示平台方人员 |
| role_id | BIGINT | 关联角色表，决定权限 |
| is_active | BOOLEAN | 账号是否启用 |

#### 3.6.4 辅助数据结构 (`sys_operation_logs` / `audit_records`)

**1. 操作日志 (`sys_operation_logs`)**
用于记录系统内的敏感操作，满足安全审计需求 (符合 NF-001 要求)。

| 字段名 | 字段类型 | 说明 |
|--------|----------|------|
| id | BIGINT | 主键，自增 |
| user_id | BIGINT | 操作人ID |
| action | VARCHAR | 操作动作 (如：商机下架、角色分配、企业停用) |
| target_id | BIGINT | 操作对象ID |
| detail | TEXT | 变更详情或操作原因 |
| ip_address | VARCHAR | 操作IP |
| created_at | DATETIME | 操作时间 |

**2. 审批记录 (`audit_records`)**
用于记录企业入驻认领等审批流程的流转历史。

| 字段名 | 字段类型 | 说明 |
|--------|----------|------|
| id | BIGINT | 主键，自增 |
| enterprise_id | BIGINT | 申请认证的企业ID |
| applicant_id | BIGINT | 申请人ID |
| status | VARCHAR | 状态变更记录 (PENDING -> VERIFIED / REJECTED) |
| auditor_id | BIGINT | 审批人ID |
| audit_reason | TEXT | 驳回原因/审核意见 |
| created_at | DATETIME | 审批操作时间 |

## 4. 非功能需求

### 4.1 性能需求
| 指标 | 要求 |
|------|------|
| 页面加载 | 核心页面（首页、商机广场）首屏加载时间不超过 3 秒 |
| 并发能力 | 支持校友会集中活动期间的高并发访问（至少支持 500 QPS） |
| 搜索响应 | 组合筛选与关键词搜索结果返回时间 ≤ 1 秒 |

### 4.2 安全需求
| 需求 | 描述 |
|------|------|
| 传输安全 | 认证接口与核心数据必须使用 HTTPS 加密传输 (违反即阻断: SR-0001) |
| 输入验证 | 所有用户输入（如商机详情、企业介绍）必须在服务端做 XSS 转义与防注入校验 (违反即阻断: SR-0005, SR-0007) |
| 权限控制 | API 必须做严格的 RBAC 鉴权，管理员接口与普通接口分离 (违反即阻断: SR-0009, SR-0010) |
| 隐私保护 | 未点击“获取联系方式”前，企业及个人的真实联系方式必须脱敏或隐藏展示 |
| 操作日志 | 敏感操作（如管理员下架商机、分配权限）必须记录操作日志，含操作人和时间 (警告级别要求) |

### 4.3 兼容性与可用性
| 需求 | 描述 |
|------|------|
| 浏览器兼容 | 推荐支持 Chrome、Edge、Firefox、Safari 最新版本 |
| 响应式设计 | 虽然本期主要面向 PC，但基础布局应考虑到一定程度的自适应缩放 |

## 5. 验收标准

### 5.1 功能验收

| 编号 | 验收项 | 验收标准 |
|------|--------|----------|
| AC-001 | 企业认领闭环 | 新用户提交“上海威蓝”的认领申请 -> 运营后台审核通过 -> 该用户自动获取企业管理员权限并可在首页看到个性化推荐 |
| AC-002 | 供需发布与撮合闭环 | 企业用户发布采购商机 -> 立即在广场可见 -> 另一企业用户点击获取联系方式 -> 发布方收到站内信提醒 |
| AC-003 | 基础字典穿透验证 | 在后台“业务品类”新增一个“测试品类” -> 前台发布商机时下拉框可立即选择 -> 广场左侧筛选栏可立即用于过滤 |
| AC-004 | 权限隔离隔离 | 普通员工登录系统，无法看到“认领企业”以外的管理入口，无法进入后台 URL |

### 5.2 非功能验收

| 编号 | 验收项 | 验收标准 |
|------|--------|----------|
| NF-001 | 安全合规 | 敏感信息传输抓包无明文；提交包含 `<script>` 脚本的简介内容被正确转义，未触发 XSS。 |
| NF-002 | 数据完整 | 删除已被引用的基础数据字典项时，系统应拦截报错，仅允许禁用。 |
